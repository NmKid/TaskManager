# PowerShell コンソールにおける文字化け（Mojibake）の調査レポート

## 1. 発生した事象
Git のコミット＆プッシュを連続して行うコマンド（`git add . && git commit ...`）を実行した際、ターミナルに文字化け（`g[N...` のような解読不能な文字列）を持つエラーメッセージが出力された。

## 2. 文字化けの根本的な原因

文字化けは、以下の「構文エラー」と「文字コード（エンコーディング）の不一致」という2つの要因が組み合わさって発生しました。

### 要因①：構文エラーの発生
実行したコマンドの中で、複数のコマンドを直列に繋ぐアンド演算子 `&&` が使用されていました。
しかし、本環境の標準ターミナルである Windows PowerShell（バージョン 5.1）では、`&&` というステートメント区切りはサポートされていません（PowerShell 7 以降でサポート）。
そのため、PowerShell が構文エラーを検知し、エラーメッセージを出力しました。

### 要因②：エンコーディングの不一致
1. **PowerShell 5.1 側の仕様**
   - PowerShell (v5.1) は、標準の出力文字コードが「Shift-JIS (cp932)」に設定されています。そのため、構文エラーのメッセージも Shift-JIS 形式の日本語で出力されました。
2. **AIエージェント側の仕様**
   - コマンドを実行し結果を受け取るエージェント側のシステムは、世界標準である「UTF-8」エンコーディングを前提としてコンソールの出力を読み取ります。
3. **文字化けの発生**
   - エージェントが、Shift-JIS で出力された日本語を「これはUTF-8である」と誤って解釈（デコード）しようとした結果、文字が壊れて解読不能な表示（Mojibake）となってしまいました。

## 3. 実施した対策（解決策）

今後の開発やデバッグ作業において、AIエージェントや他ツールが正確にエラー内容を読み取れるよう、以下の恒久的な対策を実施しました。

### ① PowerShell プロファイルの作成と設定
PowerShell の起動時に自動的に読み込まれる設定ファイル（プロファイル）を新規作成し、ターミナルの文字コード入出力を強制的に UTF-8 に固定しました。

*   **作成したファイル:** `D:\Myzk\Onedrive\Documents\WindowsPowerShell\Microsoft.PowerShell_profile.ps1`
*   **追記した設定内容:**
    ```powershell
    $OutputEncoding = [console]::InputEncoding = [console]::OutputEncoding = New-Object System.Text.UTF8Encoding
    $env:PYTHONIOENCODING = "utf-8"
    ```
    ※ 合わせて、Python の標準出力処理も UTF-8 で行われるように環境変数 `PYTHONIOENCODING` を設定しました。

### ② 実行ポリシーの変更
上記で作成したプロファイルスクリプトの実行をシステム上で許可するため、PowerShell の実行ポリシーを変更しました。
*   **実行コマンド:** `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser -Force`

## 4. 対策後の動作確認

設定完了後、意図的に同様の構文エラー（`&&`）を発生させた結果、文字化けすることなく以下のような正しい日本語フォーマットでエラーが取得・表示されることを確認しました。

> トークン '&&' は、このバージョンでは有効なステートメント区切りではありません。

以上の対策により、コンソール起因の文字化け問題は解消されました。
