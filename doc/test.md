# テスト手順 (Test Plan)

現在の `doc/next_steps.md` や実装内容を踏まえ、以下の手順で段階的にテストを進めていくプロセスです。

### 1. 動作環境・準備の確認（最初に行うこと）
まずはツールを動かすための設定がすべて完了しているか確認します。
*   [ ] **API認証情報**: `c:\Antigravity\TaskManager\config\credentials.json` が配置されているか。
*   [ ] **Gemini APIキー**: 環境変数 `GEMINI_API_KEY` （または `.env` などの設定）が設定されているか。
*   [ ] **ライブラリ**: `flet`, `google-api-python-client`, `google-auth-httplib2`, `google-auth-oauthlib`, `google-generativeai` がインストールされているか。

### 2. アプリの起動と初期認証のテスト
いよいよ動かしてみます。
*   [ ] ターミナルで `python src/main.py` を実行する。
*   [ ] Fletの画面（GUI）がエラーなく起動し、表示されるか確認する。
*   [ ] **【初回のみ】** ブラウザが立ち上がり、Googleアカウント（Tasks・Calendar）へのアクセス許可を求められるので、許可して認証を完了させる。

### 3. 「振り分け機能 (Inbox -> Lists)」のテスト
Google Tasks側のタスクを適切なリストへ移動する処理の確認です。
*   [ ] Google Tasksの公式画面を開き、**「■メモ」**（受信トレイ）にダミーのタスクをいくつか作成する。
*   [ ] ツールの画面で `振り分け (Inbox -> Lists)` の処理を実行する。
*   [ ] エラーが発生せず、タスクがそれぞれの内容に応じた**「■〇〇」**のリストに自動で移動しているか確認する。

### 4. 「AIスケジューリングと分割機能」のテスト
このツールのコア部分である、Geminiによるタスク分析とカレンダー登録の確認です。
*   [ ] Tasks内に「1時間の会議」などの短いタスクと、「アプリの要件定義から実装・テストまで」といった**大きめのタスク**（複数に分割されそうなもの）を作成する。
*   [ ] ツールの画面で `スケジュール実行 (Tasks -> Cal)` の処理を実行する。
*   [ ] **短いタスク**がカレンダーの空き時間に登録され、タスク名が `【予定済】...` に変わるか確認する。
*   [ ] **大きなタスク**が分割され、過剰分割（例: 5個以上）と判定された場合、**GUIに「承認待ち（保留）」状態で表示されるか**確認する。

### 5. 「UIでの承認」と「予定の取り消し・再登録」のテスト
手動での微調整やイレギュラーなフローの確認です。
*   [ ] （手順4で保留になったタスクに対して）GUI上の「承認」ボタン等を押し、カレンダーへの本登録が走るか確認する。
*   [ ] Google Tasks上で、カレンダー登録済みのタスク名を編集し、先頭の **`【予定済】` とメモ欄の `[Ref:EventID:...]` を消去**する。
*   [ ] この状態で再度スケジュール実行を動かし、一度登録されたタスクであっても、未登録状態として再スケジューリングされるか確認する。
